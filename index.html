Шрифты

Шрифт состоит из глифов - векторных форм каждой буквы или символа.

  • WOFF 2.0 для более новых браузеров
  • WOFF для большенства бразеров
  • TTF в устаревших браузерах
  • EOT для поддержки устаревших версий IE (до IE9)
  • Проверка используемых шрифтов
  • Разделять на поднаборы
  • Сжатие шрифтов (GZIP - стандартный компрессор)
  • Кешировать статику

Пример

@font-face {
    font-family: 'Awesome Font';
    font-style: italic;
    font-weight: 400;
    src: local('Awesome Font Italic'),
    url('/fonts/awesome-i.woff2') format('woff2'),
    url('/fonts/awesome-i.woff') format('woff'),
    url('/fonts/awesome-i.ttf') format('ttf'),
    url('/fonts/awesome-i.eot') format('eot');
    unicode-range: U+000-5FF; /* Latin glyphs */
}

Загрузки

Приоритеты загрузки

  • Наивысший (документ, CSS, шрифты)
  • Высокий (JS, XHR)
  • Средний (JS после изоборажений, изображения)
  • Низкий (JS async, defer, изображения, JS-модули)
  • Очень низкий (изображения, prefetch)

Оптимизация загрузки

  • уменьшить число блокирующих скриптов, оставить только ядро
  • основной пакет JS загружать с async
  • оставить один файл CSS в HEAD
  • все счетчики, метрики и сторонние скрипты грузим отложенно на JS

Подключение скриптов

  • Специальные атрибуты async и defer используются для того, чтобы пока грузится внешний скрипт – браузер показал остальную (следующую за ним) часть страницы. Без них этого не происходит.
  • Разница между async и defer: атрибут defer сохраняет относительную последовательность скриптов, а async – нет. Кроме того, defer всегда ждёт, пока весь HTML-документ будет готов, а async – нет.
